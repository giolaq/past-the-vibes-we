# React Native Analysis

Read the current screens, components, and data modules before writing anything. Separate facts you observed in the code from assumptions. Record which parts are portable to a TV (shared JS/TSX and data) and which need TV-specific work (navigation, remote focus). Write findings to a document; do not change app code during analysis.
